# RamanUQ v0.2.0

v0.2.0 is a demonstration-and-polish release layered on top of v0.1.0. It adds a second real-data validation case and one figure, resolves the cosmetic items v0.1.0 deferred, and tidies the repository. No pre-registered result changed, no new scientific finding is claimed, and every frozen number from v0.1.0 stands exactly as released; the headline Q1/Q2/Q3 results and their scope qualifiers ("on the spectra, configurations, and regimes tested here") are unchanged.

## What's new
- **Gate V5b (second real-data demonstration, area convention).** One additional digitized published spectrum (fCNT) reproduced by the paper's stated *area* method, within the pre-declared ±10%: reproduced I_D/I_G = 1.7136 vs. published 1.64 (window 1.476–1.804) — PASS. Together with V5 (height convention, v0.1.0), both intensity conventions are now demonstrated on real published data, not only on synthetic ground truth. [key: gates.V5b]
- **Figure F10 (replicate-averaging MDC curve).** Extends the F7 minimum-detectable-change story by showing how the detectable change in I_D/I_G shrinks as 1/√N_rep when multiple spectra per condition are averaged — the practical lever an experimenter has at fixed SNR. Derived from the committed Tier-B study results; no new study was run.

## Housekeeping (no scientific impact)
- Dependency pins refreshed/tightened in the lockfile for reproducible fresh-clone installs.
- F9 rank-stability annotation now reads its summary value from the committed results (number-injection invariant) rather than a hard-coded figure constant.
- Dead-code removal across the package; no change to any science-critical path.
- Added an explicit area-vs-height intensity-definition caveat to the protocol/limitations text.
- Resolved the cosmetic figure items v0.1.0 deferred to a follow-up patch: F3 marker-label and F1(b) reference-line label decluttering.
- Docs reorganized: provenance moved under `docs/provenance/`; per-wave briefings consolidated.

## Validation
All pre-registered validation gates pass in continuous integration on Python 3.11/Linux, unchanged from v0.1.0:
- V1, V1b, V2, V3, V4, V6 — as in v0.1.0 (no tolerance or result changed).
- V5 (height) — reproduced I_D/I_G = 1.523 vs. published 1.6 (window 1.44–1.76). [key: gates.V5]
- V5b (area, NEW) — reproduced I_D/I_G = 1.7136 vs. published 1.64 (window 1.476–1.804). Both intensity conventions now demonstrated on real data. [key: gates.V5b]

## Key results (unchanged from v0.1.0)
No new findings; every number is still generated by the pipeline into report_data.json.
- Q1 ranking is empty: zero (configuration × SNR) cells reach the pre-registered 0.90 empirical-coverage floor (maximum observed 0.80).
- Q2: standard reported 95% error bars on I_D/I_G contain the truth only 0.276 / 0.240 / 0.183 of the time at SNR 15/50/200.
- Q3: the protocol minimum-detectable-change beats the naive pipeline in all three SNR regimes (naive requires 1.407/2.819/1.245× larger defect changes to detect at SNR 15/50/200).

## Reproduce from a fresh clone
    git clone https://github.com/AvinGupta-ship-it/ramanuq.git
    cd ramanuq
    bash repro.sh
This regenerates the figures (now including F10) and report numbers from the committed study results and diffs them against the committed outputs.

## Scope and relationship to v0.1.0
v0.1.0 remains the original release of record for the pre-registered results. v0.2.0 is strictly additive demonstration-and-polish: the frozen findings, the validation gates, and the dated pre-registration are untouched. The added V5b case and F10 figure are downstream of the already-committed study artifact and change no claim.
