## What this release contains
RamanUQ is an open Python pipeline that quantifies how undisclosed Raman analysis choices (despike, baseline, lineshape, peak set, height-vs-area) affect the I_D/I_G defect metric in carbon nanomaterials. This release includes:
- The instrument + analysis package (src/ramanuq/): io, despike, baseline, lineshapes, model, fit, metrics, synth, hostile, grid, selectors, mdc, robust, reporting, viz.
- Independent clean-room reference implementations (refimpl/) of every science-critical formula, asserted equal in CI (Gate V6).
- A two-tier synthetic ground truth: in-family (Tier A) for instrument validation and adversarial out-of-family (Tier B) for all science, both intensity definitions recorded per spectrum.
- The committed Tier-B study results (data/synthetic/results/tierB_grid_results.parquet) - the canonical study artifact this release reproduces downstream of.
- Nine figures (F1-F9), the protocol card (docs/protocol.md), the disclosure checklist (docs/disclosure_checklist.md), and the full report (PDF in docs/).

## Validation
All six pre-registered validation gates pass in continuous integration on Python 3.11/Linux:
- V1 in-family noise-free recovery (<0.1% relative)
- V1b in-family bootstrap coverage ([0.90, 0.98])
- V2 baseline RMS error (<2% of reference G height per method)
- V3 end-to-end near-unbiased class on Tier-B stage-1 (mean |bias| < pre-registered tolerance)
- V4 selector-audit machinery on rigged cases (exact rank recovery)
- V5 one digitized published spectrum reproduced by the paper's stated height method, within +/-10%: reproduced I_D/I_G = 1.523 vs. published 1.6 (window 1.44-1.76) [key: gates.V5]
- V6 clean-room references agree with implementation on randomized inputs (<1e-9 analytic / <1e-6 numeric)
